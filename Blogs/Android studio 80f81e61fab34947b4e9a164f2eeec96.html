View group y view diferencias básicas

View

Son un conjunto de etiquetas basicas para mostrar informacion.

Ya sean texto , imágenes vídeos y otros elementos 

<textView
android:layout_width="wrap_content"
android:layout_height="wrap_content"
android:text="hola mundo"

/>

usan etiquetas inline, osea etiquetas que se abren y se cierran ahi mismo

  • Etiquetas
    1. <TextView>: Se utiliza para mostrar texto en la interfaz de usuario.
    1. <EditText>: Proporciona un campo de entrada de texto donde los usuarios pueden escribir o editar texto.
    1. <Button>: Representa un botón que los usuarios pueden tocar para realizar una acción.
    1. <ImageView>: Muestra una imagen o un recurso gráfico en la interfaz de usuario.
    1. <LinearLayout>: Un contenedor que organiza las vistas secuencialmente, en una fila o columna, según la orientación.
    1. <RelativeLayout>: Un contenedor que organiza las vistas en relación con otras vistas o elementos en el diseño.
    1. <ConstraintLayout>: Un contenedor que utiliza restricciones para posicionar y dimensionar las vistas de forma flexible.
    1. <ListView>: Muestra una lista de elementos desplazables en la interfaz de usuario.
    1. <RecyclerView>: Proporciona una forma más flexible y eficiente de mostrar listas de elementos desplazables.
    1. <ScrollView>: Permite desplazar el contenido dentro de un área definida en caso de que el contenido sea más grande que la pantalla.
    1. <CardView>: Proporciona un contenedor con bordes redondeados, sombras y elevación, ideal para agrupar elementos relacionados.
    1. <Spinner>: Representa un control desplegable que permite al usuario seleccionar un elemento de una lista.

Introducción

Relative layout es una forma de organizar los elementos de manera independiente, por lo tanto no tendrá ni gravety ni orientación,

RelativeLayout, es muy bueno para optimizar el rendimiento, debido a que no tenemos que hacer cosas locas para poder poder cosas diferentes con LinearLayout

FrameLayout es un group view que sirve solo para organizar por regiones y no necesariamente para toda la pantalla.

Reutilizando elementos

Podremos reutilizar elementos, solo tenemos que sacarlo q un layout aparte, y luego podemos referenciacion la etiqueta de <include layout =”recurso”> or <merge>

Las diferencias son que el include hace referencia a los elementos

Mientras que el merge los trae todos los elementos dentro del layout.

El merge debe ser incluído en el layout recurso
